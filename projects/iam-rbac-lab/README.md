# IAM Role-Based Access Control (RBAC) Lab

## Objective

Designed a secure AWS IAM access model using role-based access control and least privilege principles.

## Services Used

- AWS IAM
- AWS STS
- AWS S3
- AWS CLI

## What I Did

- Created IAM users, roles, and policies
- Implemented role assumption using AWS STS
- Tested access using both AWS Console and CLI
- Verified trust relationships and permissions
- Troubleshot access control issues

## What I Learned

This lab helped me understand how AWS manages access through layered permissions and how temporary credentials improve security.

## Key Concept

IAM requires both:
- Permission to assume a role
- Trust from the role to allow access
